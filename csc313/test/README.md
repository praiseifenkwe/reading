## üí° Study Tips

1. **Practice, Practice, Practice**: Especially K-maps and number conversions

3. **Draw Diagrams**: Visualize circuits and logic flows

5. **Check Your Work**: Verify answers with simple test cases
6. **Time Yourself**: Practice under test conditions
7. **Study Groups**: Explain concepts to others to reinforce learning

9. **Sleep Well**: Your brain needs rest to consolidate learning
10. **Stay Confident**: You've prepared well!

## ‚ö†Ô∏è Common Test Pitfalls

1. **K-Map Mistakes**:
   - Wrong Gray code ordering
   - Not wrapping around edges
   - Invalid group sizes

2. **Number System Errors**:
   - Forgetting to show work
   - Wrong 2's complement range
   - Mixing up bases

3. **Boolean Algebra**:
   - Incorrect DeMorgan's application
   - Forgetting to simplify fully
   - Wrong operator precedence

4. **Circuit Design**:
   - Incomplete truth tables
   - Wrong gate connections
   - Not verifying the design
